# --- Phase 1: Platform Foundation ---
# fastapi 0.122.0 is the earliest version that allows starlette >=0.49.1
# (its range is <0.51.0,>=0.40.0). We need >=0.49.1 to clear three
# starlette CVEs flagged by pip-audit on 2026-04-25:
#   - CVE-2024-47874 (fix in 0.40.0)
#   - CVE-2025-54121 (fix in 0.47.2)
#   - CVE-2025-62727 (fix in 0.49.1)
# Earlier fastapi versions (0.109 was current; 0.118 was an
# intermediate try) cap starlette below 0.49 and would leave
# CVE-2025-62727 unfixable. Full unit suite re-run after the bump
# confirmed no regressions.
fastapi==0.122.0
# starlette is a fastapi transitive dep but pinned explicitly so a
# fresh install cannot drift back to a vulnerable patch level. Latest
# 0.49.x within fastapi 0.122.0's <0.51.0 cap.
starlette==0.49.3
# pydantic 2.10.x is required by fastapi 0.122 — earlier versions
# (2.5.3 / 2.7.4) hit a KeyError in fastapi/_compat/v2.py:310 because
# the schema dicts they emit lack the "$ref" key that fastapi 0.122
# reads when remapping definitions for OpenAPI generation.
# pydantic-core is auto-resolved by pydantic; pinned here to lock the
# exact pair the test suite was verified against.
